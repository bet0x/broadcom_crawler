# Step 7. Assigning Roles and Permissions

As an administrator, after you complete configuring VCF Single Sign-On in the VCF Operations console, you must log in to the individual components, such as, vCenter, NSX, VCF Automation console, and the VCF Operations console and assign the required roles and permissions for users or groups.

- **Assign Service Roles in vCenter:** To know more about assigning service roles in vCenter, see [Assigning Permissions](https://techdocs.broadcom.com/us/en/vmware-cis/vsphere/vsphere/9-0/vsphere-security/vsphere-permissions-and-user-management-tasks/understanding-authorization-in-vsphere.html#GUID-74F53189-EF41-4AC1-A78E-D25621855800-en).
- **Assign service roles in NSX**: To know more about assigning service roles in NSX, see [Manage Local User Accounts](/us/en/vmware-cis/vcf/vcf-9-0-and-later/9-0/advanced-network-management/administration-guide/authentication-and-authorization/managing-local-user-accounts/manage-local-user-account.html#GUID-0467ee26-0660-4555-8759-0addd8ad9d31-en_GUID-4ADD797C-FFA7-4392-AE52-7A9A8A208063).
- **Assign service roles in VCF Operations:** To know more about assigning service roles in VCF Operations, see [Assigning Service Roles in VCF Operations](/us/en/vmware-cis/vcf/vcf-9-0-and-later/9-0/fleet-management/what-is/setting-up-sso/assigning-roles-and-permissions/assigning-service-roles-in-vmware-cloud-foundation-operations.html).
- **Assign service roles in VCF Automation**: To know more about assigning service roles in VCF Automation, see [Assigning Service Roles in VCF Automation](/us/en/vmware-cis/vcf/vcf-9-0-and-later/9-0/fleet-management/what-is/setting-up-sso/assigning-roles-and-permissions/assigning-service-roles-in-vmware-cloud-foundation-automation.html).
