# Managing VCF Single Sign-On

After you configure and start using VCF Single Sign-On, you can view the existing VCF Identity Brokers, reset VCF Single Sign-On, change an identity provider, change the deployment mode, edit the configuration of an identity provider, and add or deregister existing component configurations.

- To view configured VCF Identity Brokers, see [Overview of VCF Single Sign-On Configurations](/us/en/vmware-cis/vcf/vcf-9-0-and-later/9-0/fleet-management/what-is/managing-vmware-cloud-foundation-operations-sso/sso-overview.html#GUID-b16d246e-87f8-4db0-9d28-1d77de076e45-en_id-1387dddc-2d8f-4f65-93e8-b691a46ef084).
- To reset VCF Single Sign-On, see [Reset VCF Single Sign-On](/us/en/vmware-cis/vcf/vcf-9-0-and-later/9-0/fleet-management/what-is/managing-vmware-cloud-foundation-operations-sso/reset-vmware-cloud-foundation-sso.html#GUID-cd4f1bef-de73-4701-86d8-4b64e23a9cc1-en_id-0daa68f9-7541-463e-e9c8-1199e86a36ea).
- To change the identity provider in VCF Single Sign-On, see [Change the Identity Provider in VCF Single Sign-On](/us/en/vmware-cis/vcf/vcf-9-0-and-later/9-0/fleet-management/what-is/managing-vmware-cloud-foundation-operations-sso/change-the-identity-provider-in-vcf-sso.html#GUID-703a0b32-5c7e-4d05-b439-2411565dd283-en_id-ee786cd8-4aa2-4cb1-860b-123ebdbc7d7b).
- To change the deployment mode, see [Change the Deployment Mode in VCF Single Sign-On](/us/en/vmware-cis/vcf/vcf-9-0-and-later/9-0/fleet-management/what-is/managing-vmware-cloud-foundation-operations-sso/how-to-change-the-deployment-mode-in-vcf-sso.html#GUID-1695e2a5-998c-4e24-ae32-3d0f3a63a381-en_id-571a1ea8-870f-4ada-ad26-d284442a7cfb).
- To edit the configuration of an identity provider, see [Edit the Configuration of an Identity Provider](/us/en/vmware-cis/vcf/vcf-9-0-and-later/9-0/fleet-management/what-is/managing-vmware-cloud-foundation-operations-sso/edit-the-configuration-of-an-identity-provider.html#GUID-b256b777-d2e7-4994-8f8c-f79a56e41b2f-en_id-0c3a1cbb-59d5-4db2-b463-a83a68d82805).
- To carry out additional VCF Single Sign-On component configurations, see [Additional Component Configurations](/us/en/vmware-cis/vcf/vcf-9-0-and-later/9-0/fleet-management/what-is/managing-vmware-cloud-foundation-operations-sso/additional-configurations.html#GUID-ee9b5bf1-a101-493f-8b20-74f3b1820a4c-en_id-25c42233-86ee-4620-b6b0-0a179f59cb70).
- To deregister existing VCF Single Sign-On component configurations, see [Deregister Existing Component Configurations](/us/en/vmware-cis/vcf/vcf-9-0-and-later/9-0/fleet-management/what-is/managing-vmware-cloud-foundation-operations-sso/deregister-existing-client-configurations.html).
- To manage the deployed VCF Identity Broker appliance, see [Deploying VCF Identity Broker](/us/en/vmware-cis/vcf/vcf-9-0-and-later/9-0/deployment/deploying-a-new-vmware-cloud-foundation-or-vmware-vsphere-foundation-private-cloud-/manual-deployment-of-components-to-complete-your-vcf-platform/installing-vcf-identity-broker.html#GUID-0eafc123-11e5-4983-84bf-95d5d24c60ad-en_id-915ca638-7020-4655-d9aa-6cdc9909d6c3).
- To change the identity management of a VCF Instance managed by another VCF Instance, see [Change the Identity Management of a VCF Instance Managed by Another VCF Instance](/us/en/vmware-cis/vcf/vcf-9-0-and-later/9-0/fleet-management/what-is/managing-vmware-cloud-foundation-operations-sso/change-the-identity-management-of-a-vcf-instance-managed-by-anotehr-instance-of-vcf-operations.html#GUID-6a29860f-de4e-457c-bbe9-935af86815cc-en_id-d5fa8348-0637-4ac4-ecb2-aa2697d91eb6).
