# Connect а VCF Instance to an Existing VCF Single Sign-On Configuration

After you have configured VCF Single Sign-On, you can connect to an existing Single Sign-On configuration and add the components of the VCF Instance to the pre-configured VCF Identity Broker.

**Prerequisites**

- Ensure that you have added a VCF Instance. For more information, see [Step 1: Select a VCF Instance](/us/en/vmware-cis/vcf/vcf-9-0-and-later/9-0/fleet-management/what-is/setting-up-sso/select-a-vcf-instance.html).
- Ensure that you have configured an identity provider for VCF Single Sign-On.

**Procedure**

1. From the VCF Operations navigation pane, select Fleet ManagementIdentity & Access.
2. From the Identity & Access pane, select the VCF Instance that you want to configure to an existing SSO configuration from the VCF Instances drop- down option. For more information about selecting a VCF Instance, see [Step 1: Select a VCF Instance](/us/en/vmware-cis/vcf/vcf-9-0-and-later/9-0/fleet-management/what-is/setting-up-sso/select-a-vcf-instance.html).
3. From the Enable Single Sign-On page, click the Connect to an existing SSO configuration option and click Continue.
4. Select the identity broker instance from the Identity Broker drop down option to connect а VCF Instance to an existing VCF Single Sign-On configuration.
5. From the Configure clients page, select the VCF Instance components and click Configure.
6. From the Role Assignment Required dialog, select the I confirm that I understand the requirement to perform role assignments in order to enable SSO for the selected component(s). check box and click Continue.
