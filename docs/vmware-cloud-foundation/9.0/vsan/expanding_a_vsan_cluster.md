# Expanding a vSAN Cluster

You can expand an existing vSAN cluster by adding ESX hosts or adding devices to existing ESX hosts, without disrupting any ongoing operations.

Use one of the following methods to expand your vSAN cluster.

- Add new ESX hosts to the cluster that are configured using the supported cache and capacity devices. See [Add an ESX Host to the vSAN Cluster](/us/en/vmware-cis/vcf/vcf-9-0-and-later/9-0/vsan-deployment-administration-and-monitoring/administering-vmware-vsan/expanding-and-managing-a-vsan-cluster/expanding-a-vsan-cluster/add-a-host-to-the-virtual-san-cluster.html#GUID-627ac56e-28ee-4624-8369-efc429dd40cb-en).
- Move existing ESX hosts to the vSAN cluster and configure them by using a host profile. See [Configuring ESX Hosts in the vSAN Cluster Using Host Profiles](/us/en/vmware-cis/vcf/vcf-9-0-and-later/9-0/vsan-deployment-administration-and-monitoring/administering-vmware-vsan/expanding-and-managing-a-vsan-cluster/expanding-a-vsan-cluster/configuring-hosts-in-the-vsan-cluster-using-host-profile.html#GUID-8971daab-88b8-494d-bbf1-9fe75f7b989e-en).
- Add new capacity devices to ESX hosts that are cluster members. See [Add Devices to the Disk Group in vSAN Cluster](/us/en/vmware-cis/vcf/vcf-9-0-and-later/9-0/vsan-deployment-administration-and-monitoring/administering-vmware-vsan/device-management-in-a-vsan-cluster/working-with-individual-devices-in-vsan-cluster/add-devices-to-the-disk-group-in-vsan-cluster.html#GUID-da35aea0-f9e8-4748-a49a-698476829c34-en).
